## BAB III. KEGIATAN PELAKSAAN PRAKERIN

### 3.1 Penempatan Prakerin
Selama kegiatan Praktik Kerja Lapangan (PKL) di [Nama Perusahaan], penulis ditempatkan pada divisi IT Development sebagai *Full Stack Developer Intern*. Fokus utama penulis adalah terlibat dalam pengembangan aplikasi manajemen proyek.

Tugas dan tanggung jawab penulis meliputi:
1.  **Pengembangan Fitur:** Bertanggung jawab atas implementasi fitur *frontend* dan *backend* sesuai dengan spesifikasi yang diberikan.
2.  **Kolaborasi Tim:** Berpartisipasi aktif dalam *daily standup meeting* dan menggunakan Git untuk kolaborasi.
3.  **Quality Assurance:** Melakukan pengujian mandiri terhadap fitur yang dikembangkan.

### 3.2 Hasil PKL
Selama masa PKL, penulis berhasil menyelesaikan beberapa target capaian sebagai berikut:

**A. Aplikasi Manajemen Proyek**
Penulis berkontribusi dalam pembangunan sistem manajemen proyek yang memiliki fitur-fitur unggulan:
*   **Manajemen Proyek & Klien:** Fitur untuk mengelola data proyek dan klien secara terpusat.
*   **Manajemen Tugas (Task Management):** Sistem pembuatan dan pelacakan tugas dengan status yang dapat disesuaikan, serta dukungan import/export Excel.
*   **Time Tracker:** Fitur pencatatan waktu kerja real-time dan manual untuk memantau produktivitas tim.
*   **Dashboard Statistik:** Menampilkan ringkasan data proyek dan tugas dalam bentuk grafik.
*   **User Management:** Implementasi *Role-Based Access Control (RBAC)* untuk membatasi akses fitur berdasarkan peran pengguna (Admin, User).

**B. Teknologi yang Digunakan**
Dalam pengembangan aplikasi ini, teknologi yang diterapkan meliputi:
*   **Frontend:** React.js dengan Vite dan Tailwind CSS untuk antarmuka yang cepat dan responsif. State management menggunakan Zustand.
*   **Backend:** Laravel untuk menangani logika bisnis dan API, dengan autentikasi menggunakan Sanctum.
*   **Database:** MySQL sebagai penyimpanan data relasional.
*   **Tools:** VS Code, Git, Postman, dan Laragon/Herd.

**C. Kontribusi untuk Perusahaan**
Implementasi aplikasi ini memberikan dampak positif bagi operasional perusahaan, antara lain:
*   Mempermudah pemantauan progres proyek dan kinerja tim secara *real-time*.
*   Meningkatkan efisiensi pencatatan waktu kerja karyawan.
*   Menyediakan laporan tugas yang akurat melalui fitur export Excel.

### 3.3 Kendala Saat Pelaksanaan Prakerin
Selama pelaksanaan PKL, penulis menghadapi beberapa kendala, baik teknis maupun non-teknis:

1.  **Kendala Teknis:**
    *   **Adaptasi Framework:** Penulis memerlukan waktu untuk memahami struktur folder dan konsep *state management* pada React (Zustand) serta integrasi dengan API Laravel.
    *   **Debugging Fitur Real-time:** Kesulitan dalam menangani logika *timer* pada fitur Time Tracker agar tetap akurat saat browser di-refresh.
    *   **Konflik Git:** Terjadi beberapa kali konflik kode (*merge conflict*) saat menggabungkan pekerjaan dengan tim lain.

2.  **Kendala Non-Teknis:**
    *   **Pemahaman Alur Bisnis:** Membutuhkan waktu untuk memahami alur kerja manajemen proyek yang kompleks sebelum bisa menerjemahkannya ke dalam kode.

### 3.4 Solusi
Untuk mengatasi kendala-kendala tersebut, penulis melakukan upaya sebagai berikut:

1.  **Solusi Teknis:**
    *   Mempelajari dokumentasi resmi React, Laravel, dan Zustand, serta melihat tutorial terkait.
    *   Melakukan *testing* secara intensif dan menggunakan `console.log` atau debugging tools untuk menelusuri masalah pada *timer*.
    *   Berkomunikasi lebih aktif dengan tim sebelum melakukan *push* kode dan belajar cara menyelesaikan konflik Git dengan benar.

2.  **Solusi Non-Teknis:**
    *   Berdiskusi dengan mentor dan *Project Manager* untuk mendapatkan penjelasan detail mengenai alur bisnis aplikasi.
